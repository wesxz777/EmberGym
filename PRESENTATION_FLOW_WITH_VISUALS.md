### Slide 10: Entity Relationship Diagram (ERD)
**When:** After architecture overview  
**Duration:** 1 minute  
**What to Show:**
- Title: "Database Structure - Entity Relationship Diagram"
- Show main tables and relationships:
  - Users (center)
  - Members (connected to Users)
  - Classes (with details)
  - Bookings (connecting Members to Classes)
  - Payments (tracking transactions)
  - MembershipPlans
  - Trainers

**Simplified ERD Format:**
```
┌──────────────┐        ┌──────────────┐
│    Users     │────────│   Members    │
└──────────────┘        └──────────────┘
                              │
                    ┌─────────┼─────────┐
                    │         │         │
            ┌───────▼─┐  ┌───▼──────┐  ┌─▼─────────┐
            │Bookings │  │ Payments │  │ Training  │
            └───┬─────┘  └──────────┘  └───────────┘
                │
            ┌───▼──────┐
            │  Classes │
            └──────────┘
```

**What Member 3 Says:**
> "This is our database structure. We have Users as the main entity. Members extend Users with membership details. When members book classes, that creates a Booking record. Payments track all transactions. Everything is connected with relationships to ensure data integrity."

**Visual Design:**
- Use crow's foot notation for relationships
- Color code entity groups
- Show primary and foreign keys
- Professional ERD format
